Folder Structure:

- All components that can be re-used in the process, such as login/logout components, 
should be stored in and invoked from the "General Components" folder.

- All macros used in the process are to stored in the "Macros" folder within "General Components".

- All input files should be stored in the "Input" folder. For example, a template file to be copied or a file containing all the file paths.

File Structure:

- Test.xaml has already been created and can be used to test individual sequences without affecting the overall process.
